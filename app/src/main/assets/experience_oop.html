OOP

A basic requirement

For many years I've worked with Object Oriented Programming and have realized it's potential early in my carreer.

I find it empowering and the most important first step to be taken when developing.

A well thought object architecture can account for every possible need, avoiding rudimentary practices such as procedural and singletons, because "making something global to avoid passing it around is a code smell".

Object Oriented planning is not only good practice, but a necessity to any enterprise scale application.

I am a big fan of OOP.
